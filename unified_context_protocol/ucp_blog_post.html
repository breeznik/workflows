3. The Protocol Specification

3.1. The Universal Pointer

UCP solves the "Discovery Problem" (How does an agent find the context?) using a recursive discovery rule injected into the agent's system prompt:

Protocol Rule: "Before performing any task, assert the existence of [CONTEXT_ROOT]/boot.md. If found, execute the instructions within."

3.2. The Context Budget (Tiered Loading)

To prevent context window overflow, UCP implements a rigorous priority system:

  • P0 (Critical) Always Load: boot.md, context/MASTER.md. (Cost: ~500 tokens)
  • P1 (Task-Aware) Load based on Task: context/tech.md, knowledge/patterns.md.
  • P2 (On-Demand) Reference Only: knowledge/decisions.md, archive/.
