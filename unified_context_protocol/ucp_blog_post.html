Unified Context Protocol (UCP)

Breaking the Silos of Agentic Intelligence

UCP is a file-system specification that turns any repository into a "Multiplayer Server" for AI Agents. But more importantly, it is a declaration of independence.

1. Breaking Out of Proprietary Infrastructure

Today, your agent's memory is trapped in silos. Cursor stores it in a hidden database. Windsurf keeps it in a proprietary cloud. Copilot has its own closed index.

This creates Vendor Lock-in. If you switch tools, you lobotomize your project.

UCP changes the paradigm:

  • No Vector DB required: Your "database" is just Markdown files.
  • No Cloud dependency: Your context lives in your repo, on your disk.
  • Agent Agnostic: Start a task in generic VS Code. Refine it in Cursor. Finish it in Windsurf. The memory travels with the code.

2. The "Split-Brain" Architecture

The defining innovation of UCP is the Immutable Engine / Mutable Memory split. This architecture ensures that you can upgrade the "Intelligence" of your repo without interacting with your "Personal Data".

The Golden Rule

  • BIN Overwrite Strategy: Files in `bin/` are owned by the protocol. They are replaced entirely on update to fix bugs or improve logic.
  • CONTEXT Merge Strategy: Files in `context/` are owned by the User. Updates only *append* new sections or *merge* keys, preserving your history.

3. Kinetic State (Active Memory)

Static documentation is not enough. Agents need RAM. UCP introduces Kinetic State via `context/active/`.

This directory is the "Live Brain" of the project. It changes with every tool call, maintaining a real-time cursor of the agent's progress.

The Ecosystem

Powered by CorePack AI

UCP is the Reference Implementation (Seed Pack) for the CorePack ecosystem.

While UCP is the Protocol, CorePack AI is the Platform that manages it.

  • The Bridge (CLI): `cpackai` is the tool used to install and update UCP. It handles the complex logic of "Overwriting Bin" while "Merging Context".
  • The Brain (Registry): UCP is hosted on the CorePack Registry, allowing you to `cpackai install ucp` anywhere.

4. Conclusion

By treating context as a standardized operating system layer, UCP solves the fragmentation problem. It gives your repository a memory that persists across sessions, tools, and team members.
